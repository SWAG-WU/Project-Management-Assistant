# 每周 Vibe Coding 提醒小助手 — 需求文档

## 一、项目背景与目标

**背景**：计划每周用 vibe coding 的方式做一个小工具/项目/APP，解决自己生活或工作中的实际问题，持续积累作品集。

**目标**：
- 降低"选题难"的心理负担，让灵感来源自动化
- 通过任务拆解 + 每日提醒，保持每周项目的推进节奏
- 在离开电脑时也能通过飞书随时掌握进度

---

## 二、功能需求

### 功能 1：项目灵感收集 & 数据库

**原始需求**：搜集网络上的自动化项目/有趣的小项目/能帮助解决日常生活繁琐任务的小项目，建造数据库供参考。

**细化说明**：
- 数据来源：GitHub Trending、Product Hunt、Hacker News "Show HN"、Reddit r/SideProject、微信公众号/即刻等中文社区
- 每条记录包含字段：
  - `title`：项目名称
  - `description`：一句话描述
  - `source_url`：来源链接
  - `tags`：标签（自动化 / 效率工具 / 趣味 / AI / 生活类 等）
  - `difficulty`：预估难度（简单 / 中等 / 较难）
  - `estimated_days`：预计完成天数
  - `status`：未选用 / 已选用 / 已完成
  - `collected_at`：收录时间
- 支持按标签、难度筛选
- 每周自动抓取一次，新增条目去重后入库
- 提供一个简单的查看界面（CLI 列表 或 本地网页均可）

---

### 功能 2：任务拆解 & 每日计划分配

**原始需求**：帮我分解任务，告诉我一周里每天做什么，帮我分配好每天的工作量。

**细化说明**：
- 输入：本周选定的项目名称 + 简要描述 + 可用天数（默认 7 天）
- 输出：按天拆解的任务清单，每天 1~3 个子任务，工作量尽量均衡
- 拆解逻辑由 LLM（如 Claude / GPT）完成，prompt 中包含 vibe coding 的开发节奏（Day1 需求确认、Day2-3 核心功能、Day4-5 完善、Day6 测试、Day7 收尾/发布）
- 拆解结果保存到本地文件（JSON 或 Markdown），方便后续追踪
- 支持手动调整某天的任务后重新保存

---

### 功能 3：每日进度推送到飞书

**原始需求**：每天将任务进度报告发到飞书，让我在没有在电脑旁边时也能了解项目情况。

**细化说明**：
- 推送时机：每天固定时间（如早上 9:00 发当日任务，晚上 21:00 发进度回顾）
- 早间推送内容：
  - 今天是第几天 / 共几天
  - 今日待完成任务列表
  - 一句激励语（可选）
- 晚间推送内容：
  - 今日任务完成情况（已完成 / 未完成）
  - 整体进度百分比
  - 明日任务预告
- 推送方式：飞书自定义机器人 Webhook（最简实现）
- 消息格式：飞书卡片消息（Card Message），视觉清晰
- 任务完成状态的更新方式：
  - 方案 A（MVP）：手动在本地 JSON 文件中标记完成，脚本读取后推送
  - 方案 B（进阶）：飞书消息中带按钮，点击直接标记完成

---

## 三、技术栈建议

| 模块 | 建议方案 |
|------|----------|
| 语言 | Python 3.10+ |
| 灵感抓取 | `requests` + `BeautifulSoup` / GitHub API / RSS |
| 数据库 | SQLite（本地轻量，够用） |
| 任务拆解 LLM | Claude API 或 OpenAI API |
| 定时任务 | `schedule` 库 或系统 cron / Windows 任务计划 |
| 飞书推送 | 飞书自定义机器人 Webhook（POST JSON） |
| 配置管理 | `.env` 文件存 API Key、Webhook URL 等敏感信息 |

---

## 四、MVP 范围（第一周交付）

优先实现以下核心闭环，其余功能迭代补充：

- [x] 手动录入一个项目，LLM 自动拆解为 7 天任务计划
- [x] 任务计划保存为本地 JSON
- [x] 每天定时推送早间任务 + 晚间进度到飞书
- [ ] 灵感自动抓取（第二周迭代）
- [ ] 飞书按钮交互标记完成（第三周迭代）

---

## 五、验收标准

1. 给定一个项目描述，能在 30 秒内生成 7 天任务计划并保存
2. 飞书每天准时收到早/晚两条推送，内容格式正确
3. 进度百分比计算准确，与本地 JSON 状态一致
4. 所有 API Key 不硬编码，通过 `.env` 管理

### 功能 4：飞书移动端项目管理

**原始需求**：能够通过移动设备（飞书平台）对项目进行管理。

**细化说明**：
- 通过飞书机器人接收用户指令，支持以下操作：
  - 查看当前项目进度
  - 标记某条任务为已完成
  - 查看今日 / 明日任务
  - 切换当前进行中的项目
- 交互方式：飞书卡片消息 + 按钮回调（Interactive Message）
- 后端监听飞书事件回调，处理用户点击/输入后更新本地 JSON 状态

---

### 功能 5：移动端灵感速记

**原始需求**：有新想法时能够随时通过移动设备发送至项目文件夹中。

**细化说明**：
- 在飞书中向机器人发送一段文字，自动识别为新灵感条目
- 自动补全字段（`collected_at` 等），写入本地灵感数据库（SQLite）
- 可附加标签关键词，如"#AI #效率"，自动解析为 `tags` 字段
- 写入成功后飞书回复确认消息，包含已保存的条目摘要
