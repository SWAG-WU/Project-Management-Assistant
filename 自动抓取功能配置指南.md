# 自动抓取功能配置指南

## 📋 当前缺少的配置信息

要实现「项目灵感自动抓取」功能，需要补充以下配置：

---

## 1. GitHub API Token（推荐优先配置）

### 为什么需要？
- 抓取 GitHub Trending 项目
- 避免 API 限流（未认证每小时只能请求 60 次，认证后 5000 次）
- 获取项目详细信息（star 数、描述、语言等）

### 如何获取？

**步骤：**
1. 登录 GitHub：https://github.com
2. 点击右上角头像 → Settings
3. 左侧菜单最底部 → Developer settings
4. Personal access tokens → Tokens (classic)
5. 点击 Generate new token (classic)
6. 填写信息：
   - Note：填写 `Vibe Coding Collector`
   - Expiration：选择 `No expiration` 或 `90 days`
   - 勾选权限：只需勾选 `public_repo`（读取公开仓库）
7. 点击 Generate token
8. **立即复制 Token**（只显示一次，格式：`ghp_xxxxxxxxxxxx`）

**添加到 .env：**
```
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

---

## 4. 抓取策略配置

### 需要在 .env 中添加：

```
# 抓取频率（cron 表达式或简单描述）
COLLECT_SCHEDULE=weekly  # 每周抓取一次

# 抓取的数据源（逗号分隔）
COLLECT_SOURCES=github  # github

# GitHub Trending 语言过滤（留空表示所有语言）
GITHUB_LANGUAGES=python,javascript,typescript

# GitHub Trending 时间范围
GITHUB_TRENDING_RANGE=weekly  # daily, weekly, monthly

```

## 🚀 最小可用方案（MVP）

如果想快速启动，**只需配置 GitHub Token**：

1. 按照上面步骤获取 GitHub Token
2. 添加到 .env：`GITHUB_TOKEN=ghp_xxx`
3. 其他数据源可以后续迭代添加

这样就可以实现：
- ✅ 抓取 GitHub Trending 项目
- ✅ 自动去重并存入数据库
- ✅ 按标签和难度分类
- ⏳ Reddit/Product Hunt 等其他源后续添加

## ❓ 常见问题

**Q: GitHub Token 会过期吗？**
A: 如果选择 "No expiration" 则不会过期，但建议定期更换以保证安全。

**Q: 不配置 Token 可以抓取吗？**
A: 可以，但会受到严格的频率限制（每小时 60 次请求），且无法获取完整信息。

**Q: 需要配置所有数据源吗？**
A: 不需要，建议先配置 GitHub，其他数据源可选。
